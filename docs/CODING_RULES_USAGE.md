# 编码规则接口使用说明

## 概述

`get_coding_rules` 接口用于获取 Delphi 源码的编码规则，以便智能体能够按照规则审核和生成代码。

## 接口详情

### 工具名称
`get_coding_rules`

### 功能描述
获取 Delphi 源码编码规则。默认读取 `config/CODING_RULES.mdc` 文件，如果用户项目目录下存在 `CODING_RULES.mdc`，则使用用户自定义规则覆盖默认规则。

### 参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| project_path | string | 否 | 项目路径，用于查找用户自定义的编码规则文件 |

### 返回值

```json
{
  "success": true,
  "message": "成功获取编码规则",
  "rules": "编码规则内容...",
  "source": "default",  // 或 "user"
  "default_rules_path": "默认规则文件路径",
  "user_rules_path": "用户规则文件路径（如果存在）"
}
```

## 使用示例

### 示例 1: 获取默认编码规则

```python
# 不提供 project_path 参数，返回默认编码规则
result = await get_coding_rules()
```

### 示例 2: 获取项目编码规则

```python
# 提供 project_path 参数，优先使用项目目录下的自定义规则
result = await get_coding_rules(
    project_path="c:\\MyProject"
)
```

## 规则优先级

1. 如果提供了 `project_path` 参数且项目目录下存在 `CODING_RULES.mdc` 文件，则使用该文件中的规则。
2. 否则，使用默认规则文件 `config/CODING_RULES.mdc` 中的规则。
3. 如果两者都不存在，返回错误信息。

## 自定义规则文件

用户可以在项目目录下创建 `CODING_RULES.mdc` 文件来自定义编码规则。文件格式为 Markdown，可以包含以下内容：

```markdown
# 编码规范

## 命名规则
- 常量命名字母全大写，单词间用下划线连接
- 变量名、函数名、类型命名使用大驼峰命名法

## 代码格式化
- 按 Delphi 默认规则格式化
- 缩进使用 2 个空格

## 审核规则
- 执行内存泄露检测
- 执行异常处理检测
```

## 集成到智能体

智能体可以在以下场景中使用此接口：

1. **代码生成前**: 读取编码规则，确保生成的代码符合规范
2. **代码审核时**: 根据编码规则检查代码质量
3. **代码重构时**: 确保重构后的代码符合编码规范

## 注意事项

1. 规则文件使用 UTF-8 编码
2. 规则文件内容为纯文本，支持 Markdown 格式
3. 用户自定义规则会完全覆盖默认规则，不会进行合并
4. 如果需要部分覆盖，建议在用户规则文件中包含所有需要的规则

## 错误处理

如果获取编码规则失败，接口会返回：

```json
{
  "success": false,
  "message": "错误信息",
  "rules": ""
}
```

常见错误原因：
- 默认规则文件不存在
- 规则文件读取失败（权限问题、编码问题等）
- 提供的 project_path 无效
